# Quality Analysis Report Aggregator

Aggregates quality analysis reports across multiple RHOAI component repositories and generates combined reports with statistics and comparisons.

## Overview

This tool processes quality analysis reports generated by the `/quality-repo-analysis` skill and creates:
- Combined markdown reports with statistics  
- **Beautiful interactive HTML reports** with animated scores and collapsible sections
- JSON exports for programmatic access
- Analysis command lists for batch processing

## Repository Discovery

The tool **dynamically discovers repositories** from the `architecture-context` directory, reading component information from the latest RHOAI version. No manual configuration needed!

The aggregator automatically categorizes repositories as:
- **Upstream**: `opendatahub-io` organization
- **Downstream**: `red-hat-data-services` organization  
- **External**: Other organizations (ray-project, kubernetes-sigs, etc.)

Currently analyzing **45+ repositories** automatically discovered from `architecture-context/architecture/rhoai-3.4-ea.2/`.

## Quick Start

### 1. Generate Analysis Commands

First, generate the list of commands to run quality analysis on all repositories:

```bash
python3 aggregate_quality_reports.py --generate-commands --commands-output analysis_commands.txt
```

This creates a file with all the `/quality-repo-analysis` commands you need to run.

### 2. Run Quality Analysis

You can run these commands manually in Claude Code, or create a script to process them. For example:

```bash
# In Claude Code, run each command from analysis_commands.txt
# The reports will be generated as markdown files

# Example for a single repo:
/quality-repo-analysis https://github.com/red-hat-data-services/kserve
```

### 3. Organize Reports

Collect all generated quality report markdown files in a directory:

```bash
mkdir quality_reports
mv *_quality_report.md quality_reports/
# Or manually copy reports to this directory
```

### 4. Generate Combined Report

Process all reports and generate a combined HTML dashboard:

```bash
python3 aggregate_quality_reports.py \
  --reports-dir quality_reports \
  --output-html quality_dashboard.html
```

Then open `quality_dashboard.html` in your browser!

## Report Naming Convention

The aggregator expects report files to follow one of these patterns:
- `{repo_name}_quality_report.md`
- Any markdown file containing the repository name

Examples:
- `kserve_quality_report.md`
- `quality_analysis_kserve.md`
- `kserve_2026-04-16.md`

## Troubleshooting

### No reports found
- Check that report files are in the specified directory
- Verify file names contain the repository name
- Ensure files have `.md` extension

### Parsing errors
- Verify report follows expected markdown format
- Check for quality scorecard table with scores
- Ensure overall score is present in report

### Missing scores
- Some repositories may not have all quality dimensions
- Missing scores default to 0.0 in aggregation
- Check individual reports for completeness

### HTML not displaying correctly
- Ensure you're using a modern browser (Chrome, Firefox, Safari, Edge)
- JavaScript must be enabled for interactivity
- Refresh page to replay animations
